📋 Data Collection

100% Local Processing
TokenCount and Context Clipper do NOT collect, store, or transmit any user data. All data processing occurs locally within your browser or IDE environment.

JustinXai Labs ("we," "our," or "us") operates the TokenCount and Context Clipper browser extensions and related products. This Privacy Policy explains how we handle data for these extensions.

What We Do NOT Collect:

  • Website content you visit or interact with
  • Personal identification information
  • Usage statistics or analytics
  • Browsing history
  • Any data transmitted to external servers

Our extensions are designed with privacy as a core principle. The extensions analyze text locally on your device to provide token counting and context optimization features. No content from websites you visit—including any text, code, or data—is collected, stored, or transmitted to any external server or third party.

⚙️ Data Usage

Because we do not collect any user data, there is no data to use or process beyond what remains entirely on your local device.

Any text analysis, token counting, or context optimization performed by our extensions occurs:

  • 100% locally within your browser environment
  • Temporary memory only (not persisted)
  • No network requests for data processing

We do not use cookies, tracking pixels, or any other tracking technologies.

🔗 Third-party Disclosure

We do not sell, trade, or otherwise transfer any user data to outside parties. Because we do not collect user data in the first place, there is no data to share.

Browser Extension Permissions:

Our Chrome extension may request the following permissions for functionality:

  • activeTab — To analyze text content within the current tab for context optimization
  • storage — To save your personal settings and preferences locally

These permissions are used solely for the technical operation of the extension and do not involve any data collection or external transmission.

🔒 Data Security

Since no user data is collected or stored externally, traditional data security measures regarding server-side protection are not applicable.

However, we are committed to security best practices:

  • All processing is performed locally within your environment
  • No external network requests for user data
  • Code is reviewed for security vulnerabilities
  • Extensions are published through official marketplace channels
